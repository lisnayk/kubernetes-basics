# Самостійна робота №1: Перенесення архітектури з Docker Compose в Kubernetes

Це практичне завдання для виконання під час лекції. Ваша мета — взяти готову архітектуру на базі Docker Compose та розгорнути її в локальному кластері Kubernetes (Minikube або Docker Desktop K8s), використовуючи відповідні маніфести.

## Завдання

### 1. Перевірка локальної роботи (Docker Compose)
Перед початком роботи з Kubernetes переконайтеся, що додаток працює коректно:
1. Запустіть систему (порточна директорія github.com):
   ```bash
   docker-compose up --build
   ```
2. Перевірте доступність через браузер: [http://localhost:8080/app](http://localhost:8080/app)
3. Зупиніть систему: `docker-compose down`.

### 2. Розгортання в Kubernetes
Вам необхідно створити YAML-файли для розгортання цієї архітектури в Kubernetes.

**Вимоги до реалізації:**
- **Namespace:** Створіть окремий простір імен `lecture-mod1`.
- **Frontend:**
  - `Deployment`: Використовуйте образ `nginx:alpine` (або зберіть свій).
  - `ConfigMap`: Для зберігання значень змінних `STUDENT_NAME`, `STUDENT_GROUP`, `STUDENT_COURSE`.
  - `Service`: Типу `ClusterIP`.
- **Ingress:**
  - Налаштуйте `Ingress` ресурс, який буде проксіювати запити `/app` на сервіс `frontend`.
  - Переконайтеся, що Ingress контролер увімкнено у вашому кластері (`minikube addons enable ingress`).

Після виконання команд `kubectl apply -f ...` ви повинні отримати доступ до сторінки за адресою, яку ви вказали в Ingress (наприклад, `http://localhost/app` або через IP кластера).

## Вимоги до здачі (Moodle)

Для зарахування роботи завантажте в Moodle наступне:
1. **Всі створені YAML-файли** (Deployment, Service, Ingress, ConfigMap, Namespace).
2. **Текстовий файл `commands.txt`**, у якому перераховані всі команди, які ви виконували в терміналі для розгортання та перевірки (kubectl apply, kubectl get pods, тощо).
3. **Скріншот працюючого додатка** в браузері, де видно результат виконання (ваше ім'я, групу та курс).
