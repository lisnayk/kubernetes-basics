# Лабораторна робота №8. Пакетний менеджер Helm

## Мета роботи
Ознайомитися з пакетним менеджером **Helm** для Kubernetes. Навчитися створювати власні Helm-чарти, параметризувати маніфести та керувати релізами додатків.

## Завдання
1. Перенести додаток `k8s-monitor` (з Лабораторної роботи №7) у формат Helm-чарту.
2. Параметризувати маніфести (кількість реплік, образ, налаштування Ingress).
3. Розгорнути додаток у кластері за допомогою Helm.
4. Навчитися підключати зовнішні репозиторії та використовувати готові чарти (на прикладі Bitnami).
5. Продемонструвати можливість оновлення конфігурації через `values.yaml`.

---

## Теоретичні відомості

### Що таке Helm?
**Helm** — це пакетний менеджер для Kubernetes, який дозволяє описувати структуру додатка за допомогою шаблонів (templates) та керувати ним як єдиною сутністю (Release).

### Helm Repositories (Репозиторії)
Чарти можуть зберігатися у віддалених репозиторіях. Одним з найпопулярніших постачальників готових чартів (бази даних, веб-сервери, черги повідомлень) є **Bitnami**.

### RBAC у Helm: Role vs ClusterRole
При описі прав доступу в Helm-чартах важливо пам'ятати про область дії:
*   **Role**: Обмежена простором імен, де встановлюється реліз.
*   **ClusterRole**: Діє на рівні всього кластера. Для нашого монітора ми використовуємо саме `ClusterRole`, щоб додаток міг бачити поди у всіх просторах імен.
*   **Dynamic Namespacing**: У шаблонах Helm замість жорсткого вказання `namespace: lab7` ми використовуємо `{{ .Release.Namespace }}`, що робить чарт універсальним.

Основні команди для роботи з репозиторіями:
*   `helm repo add [назва] [url]`: Додати новий репозиторій.
*   `helm repo update`: Оновити інформацію про доступні чарти.
*   `helm search repo [назва]`: Пошук чартів у репозиторіях.

## Крок 1: Огляд структури чарту

Перейдіть до директорії з чартом:
```bash
cd labs/lab8/charts/k8s-monitor
```

Структура чарту:
*   `Chart.yaml`: Метадані про чарт (назва, версія).
*   `values.yaml`: Значення параметрів за замовчуванням.
*   `templates/`: Директорія з YAML-шаблонами.
*   `templates/_helpers.tpl`: Допоміжні функції для генерації назв та міток.

---

## Крок 2: Розгортання додатку

1. Створіть простір імен для лабораторної роботи:
   ```bash
   kubectl create namespace lab8
   ```

2. Встановіть чарт:
   ```bash
   helm install monitor ./labs/lab8/charts/k8s-monitor -n lab8
   ```

3. Перевірте статус релізу:
   ```bash
   helm list -n lab8
   kubectl get all -n lab8
   ```

4. Додайте запис у файл `hosts` (якщо використовуєте Ingress):
   `127.0.0.1 lab8.local`

5. Відкрийте `http://lab8.local` у браузері.

---

## Крок 3: Оновлення конфігурації

Однією з переваг Helm є легкість зміни параметрів.

1. Змініть кількість реплік, не редагуючи файли, а через команду:
   ```bash
   helm upgrade monitor ./labs/lab8/charts/k8s-monitor -n lab8 --set replicaCount=2
   ```

2. Перевірте, що кількість подів збільшилася:
   ```bash
   kubectl get pods -n lab8
   ```

3. Спробуйте змінити хост інгресу через власний файл значень. Створіть `my-values.yaml`:
   ```yaml
   ingress:
     hosts:
       - host: monitor.local
         paths:
           - path: /
             pathType: Prefix
   ```
   Застосуйте зміни:
   ```bash
   helm upgrade monitor ./labs/lab8/charts/k8s-monitor -n lab8 -f my-values.yaml
   ```

## Крок 4: Використання зовнішніх чартів (Bitnami)

Більшість стандартних додатків (бази даних, веб-сервери) краще встановлювати з перевірених джерел.

1. Додайте репозиторій Bitnami:
   ```bash
   helm repo add bitnami https://charts.bitnami.com/bitnami
   helm repo update
   ```

2. Знайдіть чарт для Redis:
   ```bash
   helm search repo redis
   ```

3. Встановіть Redis у простір імен `lab8`:
   ```bash
   helm install my-redis bitnami/redis -n lab8 --set architecture=standalone
   ```

4. Перевірте, які поди створив чарт Bitnami:
   ```bash
   kubectl get pods -n lab8
   ```

## Завдання для самостійної роботи

1. **Додавання параметрів**: Додайте в `values.yaml` параметр для зміни порту сервісу (зараз за замовчуванням 80). Оновіть `service.yaml` для використання цього параметра.
2. **Умовний вивід**: Додайте умову в `deployment.yaml`, щоб `resource requests/limits` додавалися лише якщо вони визначені у `values.yaml`.
3. **Bitnami MariaDB**: Встановіть MariaDB з репозиторію Bitnami. Налаштуйте пароль рута через `--set auth.rootPassword=my-password`.
4. **Bitnami MySQL**: Встановіть MySQL через чарт Bitnami. Використайте `values.yaml` для налаштування бази даних (`auth.database=app_db`) та користувача (`auth.username=app_user`).
5. **Bitnami NGINX**: Встановіть веб-сервер NGINX через чарт Bitnami. Використайте `values.yaml` для того, щоб вимкнути створення ServiceAccount (`serviceAccount.create=false`).
6. **Відкат змін**: Виконайте невдале оновлення (наприклад, вкажіть неіснуючий образ) і продемонструйте використання `helm rollback`.
